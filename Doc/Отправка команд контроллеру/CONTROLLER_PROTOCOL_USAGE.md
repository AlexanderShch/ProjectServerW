# Использование протокола контроллера (сервер)

Документ о практическом использовании протокола в `ProjectServerW`.

## Основной формат кадров

Сервер → контроллер:

`[Type][Code][DataLen][Data...][CRC16]`

Контроллер → сервер:

`[Type][Code][Status/Data][DataLen][Data...][CRC16]`

Проверка CRC обязательна для всех кадров.

## Текущее поведение сервера

- Все входящие кадры сначала помещаются в общую очередь ответа.
- Команды отправляются централизованно: одна активная команда в ожидании.
- Для активной команды выбирается ожидаемый тип/код ответа.
- Кадры «не от этой команды» переочередяются, а не удаляются.

## Особое правило `SEND_STATE`

Для `REQUEST/SEND_STATE (0x03/0x08)` допускаются два варианта подтверждения:

- стандартный response на команду;
- телеметрия `TELEMETRY/DATA_OK (0x00/0x01)`.

## Запрет авто-вызова `GET_CMD_INFO`

- `GET_CMD_INFO` (`REQUEST/0x04`) формируется только вручную из UI.
- Автоматический вызов после ошибок/таймаутов не используется.

## Практические рекомендации

- При добавлении новой команды сразу определять ожидаемый формат ответа и таймаут.
- Для команд с большим payload задавать отдельный таймаут.
- Для анализа проблем проверять последовательность: отправка → приход кадра → решение сопоставления.
