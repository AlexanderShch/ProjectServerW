# Итоги внедрения обработки команд и телеметрии

Документ фиксирует текущее рабочее состояние обмена между
сервером `ProjectServerW` и контроллером.

## Что реализовано

- Введена модель обработки команд: **single in-flight**.
- Команды отправляются только после установки соединения.
- Стартовая последовательность после подключения:
  1. `GET_VERSION`
  2. `SET_INTERVAL`
  3. `GET_DEFROST_GROUP` для групп `5` и `6`
- Регулярный `SEND_STATE` запускается только после успешного старта.

## Правила протокола

- Для `SEND_STATE` валидным подтверждением считается:
  - обычный `CommandResponse`, или
  - кадр `TELEMETRY/DATA_OK`.
- `GET_CMD_INFO` вызывается только вручную из UI.
- Для `GET_DEFROST_GROUP` используется увеличенный таймаут.

## Параметры дефроста

Поддерживаются параметры:

- `debugDisableTargetTStop`
- `debugDisableDeviceSwitchCheck`

Семантика `debugDisableDeviceSwitchCheck`:

- `0` — проверка соответствия DO->DI включена (по умолчанию)
- `1` — проверка отключена (debug)

## Синхронизация имен битов

Первая группа (DI, `T[SQ-1]`):

- `Vent1_Left`, `Vent2_Left`, `Vent1_Right`, `Vent2_Right`
- `Ten1_Left`, `Ten2_Left`, `Ten1_Right`, `Ten2_Right`
- `Vent_Out`, `Air_Open`, `Air_Close`, `Gate_Alarm`
- `Gate_Close`, `Gate_Open`, `But_Start`, `But_Stop`

Вторая группа (DO, `H[SQ-1]`):

- `_V0`, `_V1`, `_V2`, `_V3`
- `_H0`, `_H1`, `_H2`, `_H3`
- `_Out`, `_Inj`, `_Flp`, `_Opn`, `_Dbl`, `_Cls`, `_Wrk`, `_Alr`

Важно: для `DO15` используется имя `_Alr`
(не `_Stp` и не `Alrm`).

## UI и DataForm

- Состояние ПУСК/СТОП синхронизируется по `_Wrk`.
- Сообщения об останове выводятся в `labelDefrosterState`.
- Экспорт в Excel выполняется по подтвержденным условиям.
